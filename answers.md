# Вопрос 1. Задача об оптимальном префиксном коде. Метод Хаффмена

## 1. Постановка задачи

Пусть дан алфавит

[
\Lambda={a_1,a_2,\ldots,a_n},
]

где каждому символу соответствует вероятность появления

[
p(a_i), \qquad p(a_i)>0,\qquad
\sum_i p(a_i)=1.
]

Необходимо каждому символу сопоставить двоичную последовательность (код).

---

## 2. Почему недостаточно просто различных кодов

Если каждому символу поставить в соответствие разные двоичные строки, это еще **не гарантирует однозначного декодирования**.

Например

```
a → 10
b → 100
```

Получив сообщение

```
100
```

невозможно понять, это

```
a ?
```

или

```
b
```

После добавления других символов неоднозначность становится еще сильнее.

Следовательно, требуется дополнительное условие.

---

## 3. Префиксный код

### Определение

Код называется **префиксным**, если ни одно кодовое слово не является началом другого.

То есть

```
101
```

не может быть префиксом строки

```
10110
```

### Пример

Правильный код

```
a → 0
b → 10
c → 110
d → 111
```

Ни один код не начинается с другого.

Такой код можно декодировать слева направо без разделителей.

---

## 4. Длина кодового слова

Обозначим

[
l(a)
]

— длину кодового слова символа.

Средняя длина кодирования равна математическому ожиданию

[
E(l)=\sum_{a\in\Lambda} p(a)l(a).
]

Именно ее необходимо минимизировать.

---

## 5. Оптимальный префиксный код

### Определение

Оптимальным называется префиксный код, минимизирующий

[
E(l).
]

Идея очевидна:

> чем чаще символ встречается, тем короче должен быть его код.

---

# Леммы

На них основан алгоритм Хаффмена.

---

## Лемма 1. О кратчайшем префиксе

Если существует код

```
ωα
```

такой, что

```
ω
```

не является началом никакого другого кода,

то можно удалить последний символ.

Получится более короткий код, который остается префиксным.

Средняя длина уменьшится.

Следовательно,

**такой код не может быть оптимальным.**

### Что означает эта лемма

В оптимальном коде нельзя оставлять "лишние" биты.

---

## Лемма 2

Если

[
p(a)<p(b),
]

но

[
l(a)<l(b),
]

то код не оптимален.

### Идея доказательства

Меняем коды символов местами.

Изменение среднего значения

[
(p(a)-p(b))(l(a)-l(b)).
]

Оба множителя отрицательны.

Их произведение положительно.

Следовательно, после перестановки средняя длина уменьшается.

Получаем противоречие.

---

### Следствие

Самые редкие символы должны иметь самые длинные коды.

---

## Лемма о соседстве самых редких символов

Пусть

```
a
```

и

```
b
```

имеют минимальные вероятности.

Тогда существует оптимальный код, в котором

```
a → ω0
b → ω1
```

То есть

* они имеют одинаковую длину;
* отличаются только последним битом;
* являются соседними листьями дерева.

Именно эта лемма делает возможным алгоритм Хаффмена.

---

## Лемма об объединении символов

Пусть два самых редких символа

```
a
b
```

заменим одним новым символом

```
ab
```

с вероятностью

[
p(ab)=p(a)+p(b).
]

Строим оптимальный код уже для нового алфавита.

После этого код

```
ab
```

заменяем двумя

```
ω0
ω1
```

Получается оптимальный код для исходного алфавита.

Эта лемма дает рекурсивное решение задачи.

---

# Метод Хаффмена

Алгоритм построения оптимального префиксного кода.

### Шаг 1

Находим два символа с минимальными вероятностями.

---

### Шаг 2

Объединяем их в один новый символ.

Его вероятность равна сумме вероятностей.

---

### Шаг 3

Повторяем процесс.

---

### Шаг 4

Когда останутся два символа,

назначаем им коды

```
0
1
```

---

### Шаг 5

Начинаем раскрывать объединения обратно.

Если объединенный символ имел код

```
ω
```

то

```
первый → ω0
второй → ω1
```

---

## Пример

Пусть

| Символ | Вероятность |
| ------ | ----------- |
| A      | 0.40        |
| B      | 0.30        |
| C      | 0.20        |
| D      | 0.10        |

Объединяем

```
C+D
```

получаем

```
0.30
```

Теперь

```
A 0.40
B 0.30
CD 0.30
```

Объединяем

```
B+CD
```

Получаем

```
0.60
```

Далее

```
A
BCD
```

Получаем дерево

```
        *
      /   \
     A     *
          / \
         B   *
            / \
           C   D
```

Коды

```
A = 0
B = 10
C = 110
D = 111
```

---

# Почему алгоритм правильный

Основание доказательства — лемма об объединении символов.

Каждый шаг сводит задачу к такой же, но для меньшего алфавита.

В конце остается задача для двух символов, которая очевидно решается оптимально.

При обратном раскрытии оптимальность сохраняется.

Следовательно, весь алгоритм строит оптимальный префиксный код.

---

# Сложность

Если использовать очередь с приоритетом (минимальную кучу),

то

* каждое извлечение минимума — (O(\log n));
* всего выполняется (n-1) объединений.

Итого

[
O(n\log n).
]

---

# Что важно сказать преподавателю

1. Что такое префиксный код.
2. Что минимизируется средняя длина кодового слова.
3. Почему самые вероятные символы получают короткие коды.
4. Лемма о соседстве самых редких символов.
5. Лемма об объединении символов.
6. Алгоритм Хаффмена.
7. Почему он оптимален.

---

# Краткий ответ (на 2 минуты)

Оптимальный префиксный код — это префиксный код с минимальной средней длиной кодового слова. Средняя длина вычисляется как математическое ожидание длины кода. Основная идея состоит в том, что часто встречающиеся символы должны иметь короткие коды, а редкие — длинные. Доказывается, что два самых редких символа можно сделать соседними листьями дерева и объединить в один символ с суммарной вероятностью. Повторяя этот процесс, получаем алгоритм Хаффмена. После обратного раскрытия объединений строится оптимальный префиксный код. Алгоритм работает за (O(n\log n)).

# Вопрос 2. Неравенство Крафта

## Постановка задачи

Пусть заданы длины кодовых слов

[
l_1,l_2,\ldots,l_m.
]

Возникает вопрос:

> существует ли префиксный код именно с такими длинами?

Ответ дает **неравенство Крафта**.

---

## Формулировка теоремы

Для существования префиксного кода необходимо и достаточно, чтобы выполнялось

[
\boxed{\sum_{i=1}^{m}2^{-l_i}\le1.}
]

Это и называется **неравенством Крафта**.

---

## Необходимость

Представим код в виде двоичного дерева.

Каждое кодовое слово — путь от корня до листа.

Лист на глубине

[
l
]

имеет "вес"

[
2^{-l}.
]

Вес любого узла не меньше суммы весов его потомков.

Просуммировав по всему дереву,

получаем

[
1\ge
\sum2^{-l_i}.
]

Следовательно,

неравенство обязательно выполняется.

---

## Достаточность

Пусть

[
\sum2^{-l_i}\le1.
]

Тогда можно последовательно размещать листья нужных глубин в двоичном дереве.

Свободных мест всегда достаточно.

Следовательно,

существует префиксный код.

---

## Пример

Длины

```
2
2
2
```

Проверяем

[
3\cdot2^{-2}=\frac34<1.
]

Следовательно,

код существует.

Например,

```
00
01
10
```

---

Другой пример

```
1
1
1
```

Получаем

[
3\cdot\frac12=\frac32>1.
]

Такого префиксного кода быть не может.

---

## Практическое значение

Неравенство Крафта позволяет определить существование префиксного кода, **не строя его**.

---

# Что важно сказать преподавателю

* формулировку теоремы;
* смысл суммы (2^{-l_i});
* идею доказательства через двоичное дерево;
* необходимость и достаточность.

---

# Краткий ответ

Неравенство Крафта утверждает, что набор длин кодовых слов является набором длин некоторого префиксного кода тогда и только тогда, когда

[
\sum2^{-l_i}\le1.
]

Необходимость доказывается рассмотрением двоичного дерева кодов, а достаточность — построением дерева с листьями на заданных уровнях.

# Вопрос 3. Энтропия случайной схемы. Условная энтропия

## Конечная случайная схема

Пусть имеется разбиение пространства исходов

[
A_1,\ldots,A_n
]

с вероятностями

[
p_i=P(A_i).
]

Такая система называется **конечной случайной схемой (КСС)**.

---

## Собственная информация

Количество информации одного события

[
A_i
]

определяется как

[
I(A_i)=
-\log p_i.
]

Чем менее вероятно событие,

тем больше информации несет его наступление.

---

## Энтропия

### Определение

Энтропией называется математическое ожидание собственной информации:

[
\boxed{
H=-\sum p_i\log p_i.
}
]

Это формула Шеннона.

---

## Смысл энтропии

Энтропия показывает степень неопределенности случайной схемы.

* если исход заранее известен — неопределенности нет;
* если все исходы одинаково вероятны — неопределенность максимальна.

---

## Свойства энтропии

### 1.

[
H\ge0.
]

---

### 2.

[
H\le\log n.
]

Максимум достигается только при равновероятных исходах.

Доказательство основано на неравенстве Йенсена и вогнутости функции

[
f(x)=-x\log x.
]

---

### 3.

[
H=0
]

тогда и только тогда,

когда один исход имеет вероятность 1.

---

# Совместная энтропия

Для двух схем

[
\alpha,\beta
]

рассматриваются совместные события

[
A_i\cap B_j.
]

Тогда

[
H(\alpha\beta)
==============

-\sum P(A_i\cap B_j)
\log P(A_i\cap B_j).
]

---

# Условная энтропия

Пусть известно,

что произошло событие

[
A_i.
]

Тогда

[
H(\beta|A_i)
============

-\sum
P(B_j|A_i)
\log P(B_j|A_i).
]

Средняя условная энтропия

[
\boxed{
H(\beta|\alpha)
===============

\sum
P(A_i)
H(\beta|A_i).
}
]

---

# Основная формула

Используя

[
P(A_iB_j)=P(A_i)P(B_j|A_i),
]

получаем

[
\boxed{
H(\alpha,\beta)
===============

H(\alpha)
+
H(\beta|\alpha).
}
]

Это аналог формулы полной вероятности для энтропии.

---

# Свойства условной энтропии

Всегда

[
0\le
H(\beta|\alpha)
\le
H(\beta).
]

### Равенство слева

Если

[
H(\beta|\alpha)=0,
]

то после знания

[
\alpha
]

неопределенность полностью исчезает.

---

### Равенство справа

Если

[
H(\beta|\alpha)=H(\beta),
]

то знание

[
\alpha
]

ничего не сообщает о

[
\beta.
]

Это эквивалентно независимости случайных схем.

---

# Что важно сказать преподавателю

1. Определение конечной случайной схемы.
2. Определение собственной информации.
3. Формулу Шеннона.
4. Смысл энтропии как меры неопределенности.
5. Основные свойства энтропии.
6. Определение условной энтропии.
7. Формулу
   [
   H(\alpha,\beta)=H(\alpha)+H(\beta|\alpha).
   ]
8. Неравенство
   [
   0\le H(\beta|\alpha)\le H(\beta),
   ]
   и связь с независимостью.
