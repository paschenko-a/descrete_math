# Билет 22. Алгоритм Куна поиска максимального паросочетания в двудольном графе. Обоснование корректности и сложность

# 1. Постановка задачи

Пусть задан двудольный граф

$$
G=(X,Y,E),
$$

где

- \(X\) и \(Y\) — две доли графа;
- каждое ребро соединяет вершину из \(X\) с вершиной из \(Y\).

Необходимо найти **максимальное паросочетание**, то есть паросочетание, содержащее наибольшее возможное число ребер.

---

# 2. Идея алгоритма Куна

Алгоритм Куна строит паросочетание постепенно.

Сначала паросочетание пусто.

Затем для каждой вершины первой доли

$$
X
$$

пытаются найти **увеличивающую цепь**.

Если такая цепь найдена,

паросочетание увеличивается на одно ребро.

Если увеличить паросочетание невозможно,

оно уже является максимальным (по теореме Бержа).

---

# 3. Основная идея

Пусть вершина

$$
u\in X
$$

еще свободна.

Попытаемся соединить ее с некоторой вершиной

$$
v\in Y.
$$

Если

$$
v
$$

тоже свободна,

добавляем ребро

$$
(u,v).
$$

Если же

$$
v
$$

уже занята,

то пытаемся **переназначить**

вершину,

которая сейчас соединена с

$$
v,
$$

на другую вершину.

Если это удается,

то вершина

$$
v
$$

освобождается,

и ребро

$$
(u,v)
$$

можно включить в паросочетание.

Таким образом рекурсивно строится увеличивающая цепь.

---

# 4. Используемые структуры данных

Обычно хранятся:

- массив

$$
match[v],
$$

который показывает,

с какой вершиной первой доли соединена вершина

$$
v\in Y;
$$

- массив

$$
used[u],
$$

который не позволяет посещать одну и ту же вершину несколько раз при одном поиске.

---

# 5. Поиск увеличивающей цепи

Для каждой вершины

$$
u\in X
$$

запускается поиск в глубину (DFS).

Во время обхода рассматриваются все соседи вершины.

Для каждой вершины

$$
v
$$

возможны два случая.

---

### Случай 1

Вершина

$$
v
$$

свободна.

Тогда выполняем

$$
match[v]=u.
$$

Паросочетание увеличилось.

---

### Случай 2

Вершина

$$
v
$$

уже занята.

Пусть

$$
match[v]=u'.
$$

Пытаемся рекурсивно найти новое место для вершины

$$
u'.
$$

Если это удалось,

то освобождаем

$$
v
$$

и соединяем ее с вершиной

$$
u.
$$

---

# 6. Алгоритм Куна

## Шаг 1

Паросочетание пусто.

---

## Шаг 2

Для каждой вершины множества

$$
X
$$

очищается массив посещенных вершин.

---

## Шаг 3

Запускается поиск увеличивающей цепи (DFS).

---

## Шаг 4

Если цепь найдена,

размер паросочетания увеличивается на единицу.

---

## Шаг 5

После обработки всех вершин получаем максимальное паросочетание.

---

# 7. Псевдокод

```text
для каждой вершины u из X

    очистить used

    try_kuhn(u)


try_kuhn(u)

если u уже посещалась

    вернуть false

отметить u посещенной

для каждого соседа v

    если v свободна

        match[v]=u

        вернуть true

    если try_kuhn(match[v])

        match[v]=u

        вернуть true

вернуть false
```

---

# 8. Пример работы

Рассмотрим граф

```
X              Y

A -------- 1

A -------- 2

B -------- 2

B -------- 3

C -------- 3
```

---

### Шаг 1

Берем вершину

```
A.
```

Свободна вершина

```
1.
```

Получаем

```
A — 1
```

---

### Шаг 2

Берем вершину

```
B.
```

Свободна вершина

```
2.
```

Получаем

```
A — 1

B — 2
```

---

### Шаг 3

Берем вершину

```
C.
```

Свободна вершина

```
3.
```

Получаем

```
A — 1

B — 2

C — 3
```

Получено совершенное паросочетание.

---

# 9. Более интересный пример

Пусть граф имеет вид

```
A → 1

A → 2

B → 1
```

После обработки

```
A
```

имеем

```
A — 1
```

Теперь рассматриваем

```
B.
```

Вершина

```
1
```

занята.

Алгоритм пытается перенести

```
A
```

на вершину

```
2.
```

Это возможно.

Получаем

```
A — 2

B — 1
```

Размер паросочетания увеличился.

Именно такой процесс называется поиском увеличивающей цепи.

---

# 10. Обоснование корректности

Корректность алгоритма основана на **теореме Бержа**.

Каждый успешный запуск DFS находит увеличивающую цепь.

После замены ребер на этой цепи

размер паросочетания увеличивается ровно на единицу.

Алгоритм повторяет этот процесс,

пока увеличивающие цепи существуют.

Когда очередной поиск не находит увеличивающей цепи,

по теореме Бержа текущее паросочетание уже является максимальным.

Следовательно,

алгоритм всегда строит максимальное паросочетание.

---

# 11. Теорема о корректности

## Формулировка

Алгоритм Куна всегда находит максимальное паросочетание в двудольном графе.

---

## Доказательство

После каждого успешного поиска DFS

находится увеличивающая цепь.

По теореме Бержа

после замены ребер

размер паросочетания увеличивается на единицу.

Когда ни для одной вершины первой доли

увеличивающая цепь не существует,

алгоритм завершается.

По теореме Бержа

отсутствие увеличивающих цепей означает,

что текущее паросочетание является максимальным.

Следовательно,

алгоритм корректен.

---

# 12. Сложность алгоритма

Пусть

- число вершин

$$
|V|;
$$

- число ребер

$$
|E|.
$$

Для каждой вершины первой доли

запускается DFS.

Один поиск в глубину просматривает не более

$$
O(E)
$$

ребер.

Поэтому общая сложность

$$
O(VE).
$$

---

Если обозначить размеры долей

через

$$
|X|
\quad\text{и}\quad
|Y|,
$$

то обычно пишут

$$
O(|X|\cdot E),
$$

так как DFS запускается только из вершин первой доли.

---

# 13. Достоинства алгоритма

- простая реализация;
- легко программируется;
- эффективно работает на небольших графах;
- основан на красивой теории увеличивающих цепей.

---

# 14. Недостатки

- для больших графов работает медленнее алгоритма Хопкрофта—Карпа;
- сложность выше оптимальной;
- многократно запускает DFS.

---

# 15. Сравнение алгоритмов

|Алгоритм|Назначение|Сложность|
|---------|----------|----------|
|Кун|Максимальное паросочетание|$$O(VE)$$|
|Хопкрофт—Карп|Максимальное паросочетание|$$O(E\sqrt V)$$|
|Эдмондс–Карп|Максимальный поток|$$O(VE^2)$$|

# Краткий ответ (2–3 минуты)

Алгоритм Куна предназначен для поиска максимального паросочетания в двудольном графе. Он начинает с пустого паросочетания и последовательно пытается добавить в него каждую вершину первой доли. Для этого с помощью поиска в глубину строится увеличивающая цепь. Если конечная вершина свободна, ребро добавляется. Если занята, алгоритм рекурсивно пытается переназначить уже сопоставленную вершину.

Каждый успешный поиск увеличивает размер паросочетания на единицу. Когда увеличивающих цепей больше нет, по теореме Бержа текущее паросочетание является максимальным. Следовательно, алгоритм Куна всегда находит максимальное паросочетание.

При запуске DFS для каждой вершины первой доли общая сложность алгоритма составляет

$$
O(VE).
$$
