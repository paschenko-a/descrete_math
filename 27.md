# Билет 27. Остовные деревья. Алгоритмы Прима и Краскала

# 1. Остов графа

## Определение

Пусть дан связный граф

$$
G=(V,E).
$$

**Остовом** графа называется подграф,

который:

- содержит все вершины исходного графа;
- является деревом.

Иначе говоря,

остов соединяет все вершины графа,

не содержит циклов

и имеет

$$
|V|-1
$$

ребро.

---

## Пример

Исходный граф

```
A ----- B
|\      |
| \     |
|  \    |
|   \   |
D ----- C
```

Одним из остовов является

```
A ----- B
|
|
D ----- C
```

---

# 2. Остовное дерево минимального веса

Во многих задачах ребра имеют веса.

Пусть каждому ребру

$$
e
$$

соответствует вес

$$
w(e).
$$

## Определение

**Минимальным остовным деревом (Minimum Spanning Tree, MST)**

называется остов,

имеющий минимальную суммарную стоимость

$$
\sum_{e\in T}w(e).
$$

---

## Применения

Минимальные остовные деревья используются при:

- проектировании дорог;
- построении компьютерных сетей;
- проектировании линий электропередачи;
- строительстве трубопроводов;
- кластеризации данных;
- обработке изображений.

---

# 3. Свойства минимального остова

Минимальное остовное дерево:

- содержит все вершины графа;
- содержит ровно

$$
|V|-1
$$

ребер;
- является связным;
- не содержит циклов;
- имеет минимальную суммарную стоимость.

Если веса ребер различны,

то минимальный остов единственный.

---

# 4. Алгоритм Краскала

## Идея алгоритма

Алгоритм Краскала

строит дерево,

постепенно добавляя ребра

в порядке возрастания их веса.

При этом

добавляются только те ребра,

которые не образуют цикл.

---

# 5. Шаги алгоритма Краскала

1. Отсортировать все ребра по возрастанию веса.
2. Начать с пустого множества ребер.
3. Последовательно рассматривать ребра.
4. Если очередное ребро не образует цикл,
   добавить его.
5. Если цикл возникает —
   пропустить ребро.
6. Завершить работу,
   когда выбрано

$$
|V|-1
$$

ребро.

---

# 6. Псевдокод алгоритма Краскала

```text
отсортировать ребра по весу

T = ∅

для каждого ребра

    если добавление ребра
    не образует цикл

        добавить ребро

вернуть T
```

---

# 7. Пример работы алгоритма Краскала

Пусть граф имеет вид

```
A ----- B (2)

| \

|  \

4   1

|    \

D ----- C

     3
```

Порядок рассмотрения ребер:

```
1

2

3

4
```

Выбираются ребра весов

```
1

2

3
```

Получаем минимальное остовное дерево.

---

# 8. Проверка образования цикла

Для проверки,

образует ли новое ребро цикл,

обычно используется структура данных

**Система непересекающихся множеств**

(**Disjoint Set Union, DSU**, или **Union-Find**).

Она поддерживает операции:

- Find — определить компоненту вершины;
- Union — объединить две компоненты.

Обе операции выполняются практически за константное время.

---

# 9. Сложность алгоритма Краскала

Основное время занимает сортировка ребер.

Она требует

$$
O(E\log E).
$$

Операции DSU имеют почти константную стоимость,

поэтому общая сложность алгоритма

$$
\boxed{O(E\log E).}
$$

---

# 10. Алгоритм Прима

## Идея алгоритма

Алгоритм Прима

строит минимальное дерево,

начиная с одной вершины.

На каждом шаге

к уже построенному дереву

добавляется ближайшая вершина,

соединенная ребром минимального веса.

---

# 11. Шаги алгоритма Прима

1. Выбрать произвольную начальную вершину.
2. Поместить ее в дерево.
3. Среди всех ребер,
   соединяющих дерево
   с остальными вершинами,
   выбрать ребро минимального веса.
4. Добавить новую вершину.
5. Повторять,
   пока все вершины не войдут в дерево.

---

# 12. Псевдокод алгоритма Прима

```text
выбрать стартовую вершину

пока не все вершины добавлены

    выбрать минимальное ребро,
    выходящее из дерева

    добавить вершину
```

---

# 13. Пример работы алгоритма Прима

```
A ----- B (2)

| \

|  \

4   1

|    \

D ----- C

     3
```

Начинаем с вершины

```
A.
```

Добавляем последовательно ребра

```
A-C

A-B

C-D
```

Получаем минимальный остов.

---

# 14. Корректность алгоритма Прима

Алгоритм основан на следующем свойстве.

## Свойство безопасного ребра

Пусть множество вершин уже входит в строящийся остов.

Тогда ребро минимального веса,

соединяющее это множество

с остальными вершинами графа,

обязательно принадлежит некоторому минимальному остову.

Поэтому добавление такого ребра

никогда не приводит к ошибке.

---

# 15. Корректность алгоритма Краскала

Алгоритм Краскала также использует свойство безопасного ребра.

Если выбрать

самое легкое ребро,

соединяющее две различные компоненты,

то существует минимальный остов,

содержащий это ребро.

Следовательно,

каждое выбранное ребро можно безопасно включать в решение.

---

# 16. Теорема о корректности

## Формулировка

Алгоритмы Прима и Краскала

всегда строят минимальное остовное дерево.

---

## Идея доказательства

Оба алгоритма

на каждом шаге выбирают

**безопасное ребро**,

которое обязательно входит

хотя бы в один минимальный остов.

Следовательно,

после

$$
|V|-1
$$

шагов

получается дерево минимального веса.

---

# 17. Сложность алгоритма Прима

При использовании бинарной кучи

сложность составляет

$$
O(E\log V).
$$

Если используется матрица смежности,

то сложность равна

$$
O(V^2).
$$

---

# 18. Сравнение алгоритмов

|Алгоритм|Идея|Сложность|
|---------|----|----------|
|Краскал|Добавляет ребра по возрастанию веса|$$O(E\log E)$$|
|Прим|Постепенно расширяет дерево|$$O(E\log V)$$|

---

# 19. Когда какой алгоритм лучше?

**Алгоритм Краскала**

лучше подходит,

если граф разреженный,

так как сортировка небольшого числа ребер выполняется быстро.

**Алгоритм Прима**

обычно эффективнее

для плотных графов,

особенно при использовании матрицы смежности.

# Краткий ответ (2–3 минуты)

Остовным деревом называется подграф связного графа, содержащий все вершины и не имеющий циклов. Если ребра имеют веса, то минимальным остовным деревом называется остов с минимальной суммой весов.

Алгоритм Краскала сортирует все ребра по весу и последовательно добавляет их, если они не образуют цикл. Для проверки циклов используется структура DSU. Сложность алгоритма составляет

$$
O(E\log E).
$$

Алгоритм Прима начинает с одной вершины и постепенно расширяет дерево, каждый раз выбирая ребро минимального веса, соединяющее уже построенную часть дерева с новой вершиной. При использовании бинарной кучи его сложность равна

$$
O(E\log V).
$$

Корректность обоих алгоритмов основана на свойстве безопасного ребра: минимальное ребро, пересекающее любой разрез графа, обязательно входит хотя бы в одно минимальное остовное дерево.
